[1] Why would we want to have different element types for different vectors?
because the utilities vector provides can be useful to any data types: whether we are parsing sequential data streams in text (from a survey) or numbers (like weather data), it's useful to define a general purpose container.

[2] What is a template?
mechanism to use types as parameters for a class or function
The compiler inferrs a specific class or function when we pass certain arguments
also known as a type generator


[3] What is generic programming?
simplest: using templates
actually: writing code that works with a variety of types presented as arguments, as long as those argument types meet specific syntactic and semantic requirements

[4] How does generic programming differ from object-oriented programming?
focus is more the alogrithms than the data types they use
the choice of function invoked when you use generic programming is determined by the compiler at compile time, whereas oop is determined at run time

parametric polymorphism		relying on explicit template parameters	(compile-time resolution)
ad hoc polymorphism		using class hierarchies and virtual functions (aka oop)	(runtime resolution)

[5] What is a concept?
a set of requirements on a set of template arguments, indicted with the keyword "requires"

[6] What benefits do we get from the use of concepts?
Ind order to use the unsurpassed flexibility and performance of Templates with guarantees, we need concepts to ensure that all pre- and postconditions are met properly.

[8] How does resize() differ from reserve()?
Resize() changes the number of elements in a vector while reserve gets the memory to store those elements.

[9] What is a resource? Define and give examples.
something that is aquired and must be realeased (given back)
examples are: (like using new and delete on the free-store)
	-Memory
	-Locks
	-File handles
	-Thread Handles
	-Sockets
	-Windows
[10] What is a resource leak?
type of resource consumption by a program where the acquired resources are not released.

[11] List the three resource-management guarantees.
basic		succeed or throw and exception without leaking resources
strong		basic guarantee + all values not local to function (observable) remain 
		the same after failure
no-throw	beware of simple returns that leave required resources unreleased.
		will not throw and is used for simple operations

[12] How can the use of a built-in pointer lead to a resource leak? Give examples.
we can forget to delete them.
it's better to transer ownership with the move constructor
we can return an object instead of a pointer

[13] What is RAII? What problem does it address?
Resource acquisition is initialization
The resources are acquired by constructors and released by the matching destructors.
It adresses the "exception problem", there is no memory/resource leak.

[14] What is unique_ptr good for?
it's used to indicate that the returned pointer represents ownership (must be deleted)
they are created using std::make_unique
when the unique_ptr is destroyed, it deletes the object it points to
you can never have two unique pointers pointing to the same object

[15] What is shared_ptr good for?
guarantees deletion AND can be copied.
more heavyweight implementation, ensures taht the last copy destruction also destroyed the object referred to

be careful of circular references! loop must be broken using weak_ptr
