[5] What is a hash function?
Takes a given input and scrambles it using a certain algorithm like SHA-256
The output will be deterministic and ideally irreversible, this is useful
to prevent the same input generating the same output

[6] What is the basic ordering property of binary tree?
every node has exactly 2 children or none and is considered a leaf
The nodes can be red or black

[7] What (roughly) does it mean for a tree to be balanced?
When each subtree has approximately as many nodes as every other subtree that's equally far from the root

[8] How much space per element does a map take up?
- The data (std::pair<key, value>
- Parent pointer (8 bytes)
- Left Child pointer (8 bytes)
- Right Child pointer (8 bytes)
- Color bit (typically 1-8 bytes)

[9] How much space per element does a vector take up?
a std::vector<std::pair<int,int>> takes up 8 bytes for key+value per element.
The total overhead of vector is 24 bytes

The cost of lookup is near constant thanks to the hash table providing a clear lookup position

[19] What containers does the STL offer?
Vector			A contigously allocated sequence of elements; 
			use it as default container
List			a doubly-linked list; 
			use when you need to insert and delete elements without moving existing ones
Forward_list		a singly-linked list; 
			use for lists that are mostly empty
deque			a cross between a list and a vector; 
			don't use until you have expert-level knowledge of algorithms and machine architecture
map			a balanced ordered tree;
			use when you need to access elements by value
multimap		a balanced ordered tree where there can be multiple copies of a key;
			use it when you need to access elements by value
unordered_map		a hash table; an optimized version of map;
			use for large maps when you need high performance and can devise a good hash function
unordered_multimap	a hash table where there can be multiple copies of a key; an optimized version of multimap; 
			use for large maps when you need high performance and can devise a good hash function
set			a balanced ordered tree;
			use it when you need to keep track of individual values
multiset		a balanced ordered tree where there can be multiple copies of a key; 
			use it when you need to keep track of individual values
unordered_set		like unordered_map;
			but just with values, not (key,value) pairs
unordered_multiset	like unordered_multimap, but just with values, not (key,value) pairs
